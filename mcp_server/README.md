# CareBridge

A multi-agent customer support assistant for Apple. Built with LangChain, LangGraph, and Streamlit — it lets support executives ask questions in plain English and get answers pulled from the right source automatically.

## Architecture

CareBridge uses a LangGraph graph to route each query to the appropriate agent:

```
User query
    |
  Router (LLM classifies intent)
    |
  ----------------
  |       |       |
 SQL     RAG    BOTH
  |       |       |
  |       |   SQL + RAG
  |       |       |
  |       |  Synthesiser
  |       |       |
  └───────┴───────┘
       Response
```

**SQL agent** — handles questions about specific customers, ticket history, account details, and anything that requires querying structured records. Uses LangChain's SQL agent against a SQLite database.

**RAG agent** — handles questions about Apple policies by searching a ChromaDB vector store built from uploaded PDFs. Uses `MultiQueryRetriever` to improve retrieval accuracy by generating multiple rephrasings of each query.

**Router** — an LLM call that classifies each query as `sql`, `rag`, or `both`. When the route is `both`, a second LLM call strips customer-specific details from the query before hitting the vector store, so the retrieval is based on the underlying policy question rather than the customer's name.

**Synthesiser** — used only on the `both` path. Takes the SQL result and the RAG result and writes a combined answer grounded in both the customer's actual data and the relevant policy.

**MCP server** — wraps all three agents as callable tools over stdio so they can be used by any MCP-compatible client.

## Project structure

```
agents/
  sql_agent.py         natural language queries against SQLite
  rag_agent.py         policy document retrieval via ChromaDB
  graph.py             LangGraph router, agents, and synthesiser
mcp_server/
  server.py            MCP tool definitions
ui/
  app.py               Streamlit chat interface
data/
  pdfs/                Apple policy PDFs (add your own)
  chroma_db/           vector store, auto-generated by rag_pipeline.py
  apple_support.db     SQLite database, auto-generated by generate_apple_data.py
generate_apple_data.py creates synthetic customer and ticket data
rag_pipeline.py        chunks, embeds, and indexes PDFs into ChromaDB
setup_project.py       creates folders and checks dependencies
```

## Setup

**Requirements:** Python 3.10+, an OpenAI API key.

```bash
git clone https://github.com/your-username/carebridge.git
cd carebridge
python -m venv venv && source venv/bin/activate
pip install -r requirements.txt
cp .env.example .env
```

Open `.env` and add your OpenAI API key:

```
OPENAI_API_KEY=sk-...
```

Run the setup script to create all required folders and verify dependencies:

```bash
python setup_project.py
```

## Data preparation

**Customer database** — generates 100 synthetic Apple customers and 300 support tickets into a SQLite database:

```bash
python generate_apple_data.py
```

**Policy documents** — download Apple's public policy PDFs and place them in `data/pdfs/`. Recommended files:

- `apple-privacy-policy.pdf` — from apple.com/legal/privacy
- `apple-refund-policy.pdf` — from apple.com/shop/help/returns_refund
- `applecareplus.pdf` — from apple.com/legal/sales-support/applecare
- `apple-accessibility-policy.pdf` — from apple.com/ca/contact/pdf/accessibility_standards_policy.pdf

Then index them into ChromaDB:

```bash
python rag_pipeline.py
```

## Running the app

```bash
streamlit run ui/app.py
```

The app opens at `http://localhost:8501`.

## Usage

Type any question into the chat input. The system routes it automatically:

| Question type | Example | Route |
|---|---|---|
| Customer lookup | "Give me a profile of Brandon Davis" | SQL |
| Policy question | "What does AppleCare cover for accidental damage?" | RAG |
| Combined | "Does Jason Morris qualify for a refund?" | BOTH |

You can also upload additional policy PDFs directly from the sidebar — they are indexed immediately and become searchable in the same session.

Each response shows a route badge (SQL / RAG / BOTH) and, for policy answers, the source document the answer was drawn from.

## Environment variables

| Variable | Default | Description |
|---|---|---|
| `OPENAI_API_KEY` | — | Required. Your OpenAI API key. |
| `LLM_MODEL` | `gpt-4o` | Model used for the agents and router. |
| `EMBEDDING_MODEL` | `text-embedding-3-small` | Model used for PDF embeddings. |
| `DB_PATH` | `data/apple_support.db` | Path to the SQLite database. |
| `CHROMA_DIR` | `data/chroma_db` | Path to the ChromaDB vector store. |
